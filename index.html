📖 Git이란?

Git은 버전 관리 시스템(Version Control System)입니다. 프로젝트의 변경사항을 기록하고, 이전 버전으로 되돌릴 수 있으며 여러 사람이 동시에 협업할 수 있도록 도와줍니다.

Git의 장점

  • 변경 이력 관리
  • 여러 명이 동시에 협업 가능
  • 브랜치를 이용한 독립적인 개발
  • 실수해도 이전 버전으로 복구 가능
  • GitHub와 함께 사용 가능

Windows

https://git-scm.com

macOS

brew install git
또는 
xcode-select --install

설치 확인

git --version

🌿 Git 기본 개념

용어 설명
Repository 프로젝트 저장소
Commit 변경사항 저장
Branch 독립적인 작업 공간
Merge 브랜치 합치기
Remote 원격 저장소(GitHub)

⌨️ 자주 사용하는 명령어

저장소 생성

git init

복제

git clone URL

상태 확인

git status

파일 추가

git add .

커밋

git commit -m "메시지"

로그 보기

git log --oneline --graph

브랜치 생성

git branch feature/login

브랜치 이동

git switch feature/login

병합

git merge feature/login

원격 업로드

git push origin main

최신 내용 가져오기

git pull origin main

❓ FAQ

Git과 GitHub는 같은 것인가요?

아닙니다. Git은 버전관리 프로그램이고 GitHub는 Git 저장소를 온라인에서 관리하는 서비스입니다.

commit과 push 차이는?

commit은 내 컴퓨터에 저장하는 것이고, push는 GitHub 같은 원격 저장소에 업로드하는 것입니다.

branch는 왜 사용하나요?

기존 코드를 망가뜨리지 않고 새로운 기능을 개발하기 위해 사용합니다.
